# C2CAD-Bench Data Card

## Files

| File | Description |
| --- | --- |
| `data/cases.jsonl` | 75 benchmark prompts and deterministic golden references. |
| `data/model_outputs.jsonl` | Raw normalized model outputs and per-case scores for 13 models. |
| `data/scores.csv` | Tabular model-case score table. |
| `data/model_summary.csv` | Aggregate model-level score summary. |
| `data/prompt_audit.csv` | Machine-generated prompt scaffolding audit. |
| `results/showcase_db.js` | Static result database used by the viewer. |

## Record Structure

Each case record contains:

- `case_id`
- `family`
- `phase`
- `difficulty_id`
- `difficulty_label`
- `expected_shape_count`
- `prompt`
- `golden_shapes_json`

Each model-output record contains:

- `case_id`
- `model_id`
- family and difficulty metadata
- output primitive count
- coverage, geometry, semantic, and global scores
- `output_shapes_json`

## Collection Method

Prompts and golden references are generated by deterministic Python family generators in `stages/`. Model outputs were collected by querying provider APIs through `runners/run_unified.py` and scoring outputs against the released references.

## Privacy and Sensitive Data

The dataset contains synthetic CAD-style prompts, generated JSON outputs, and score tables. It does not contain human-subject data, personal data, or scraped user content.

## Limitations

The data covers primitive assembly reasoning, not full CAD design. The current release includes prompts with varying degrees of scaffolding. See `data/prompt_audit.csv` and `docs/known_limitations.md`.
